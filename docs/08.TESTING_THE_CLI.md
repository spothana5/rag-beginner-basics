# 07. Testing the CLI — Running Your RAG Pipeline

Now that everything is built, it's time to actually **run** your RAG pipeline from the terminal. There are two commands: `ingest` (load documents) and `query` (ask questions).

---

## Prerequisites

Before running these commands, make sure:

1. **Virtual environment is activated**:
   ```bash
   source .venv/bin/activate
   ```
   Verify with `which python` — it should point to `.venv/bin/python`, not `/opt/homebrew/...`

2. **OpenAI API key is set** in your `.env` file:
   ```bash
   cp .env.example .env
   # Then edit .env and add your actual API key
   ```

---

## Step 1: Ingest Documents

This command reads your sample text files, chunks them, converts them to embeddings, and stores them in ChromaDB.

```bash
python -m rag_beginner_basics ingest --data-dir ./data/sample_texts
```

### What happens behind the scenes:

```
1. __main__.py          → Entry point, calls main()
2. main.py              → Parses arguments: command="ingest", data_dir="./data/sample_texts"
3. load_dotenv()        → Loads OPENAI_API_KEY from .env file
4. RAGPipeline.ingest() → Runs the full ingestion pipeline:
   a. document_loader.py  → Reads all 4 .txt files into Document objects
   b. chunker.py           → Splits each document into ~500 character chunks
   c. embedder.py          → Sends each chunk to OpenAI API → gets 1536-dimensional vectors
   d. vector_store.py      → Stores chunks + vectors in ChromaDB (saved to ./vectordb_data/)
5. Prints: "Ingested X chunks into collection 'rag-basics'"
```

### Expected output:

```
Ingested 24 chunks into collection 'rag-basics'
```

(The exact number depends on your document sizes and chunk settings.)

### Optional flags:

| Flag | Default | Description |
|------|---------|-------------|
| `--data-dir` | **(required)** | Path to the folder containing your documents |
| `--pattern` | `*.txt` | File pattern to match (e.g., `*.pdf` for PDF files) |
| `--chunk-size` | `500` | Number of characters per chunk |
| `--overlap` | `50` | Number of overlapping characters between chunks |
| `--collection` | `rag-basics` | ChromaDB collection name |
| `--persist-dir` | `./vectordb_data/chromadb` | Where ChromaDB saves data on disk |

### Example with custom settings:

```bash
python -m rag_beginner_basics ingest --data-dir ./data/sample_texts --chunk-size 300 --overlap 30
```

## Step 2: Query Your Documents

Once documents are ingested, you can ask questions and get answers grounded in your data.

```bash
python -m rag_beginner_basics query "What is machine learning?"
```

### What happens behind the scenes:

```
1. __main__.py          → Entry point, calls main()
2. main.py              → Parses arguments: command="query", question="What is machine learning?"
3. load_dotenv()        → Loads OPENAI_API_KEY from .env file
4. RAGPipeline.query()  → Runs the full query pipeline:
   a. embedder.py          → Converts your question into a 1536-dimensional vector
   b. vector_store.py      → Searches ChromaDB for the 3 most similar chunks
   c. rag_pipeline.py      → Builds a prompt with retrieved chunks as context
   d. OpenAI GPT           → Generates an answer based ONLY on the provided context
5. Prints: Question, Answer, and Source files
```

### Expected output:

```
Question: What is machine learning?

Answer: Machine learning is a subset of artificial intelligence that enables
computers to learn from data and improve their performance on tasks without
being explicitly programmed...

Sources: machine_learning.txt, machine_learning.txt, machine_learning.txt
```

### Optional flags:

| Flag | Default | Description |
|------|---------|-------------|
| `question` | **(required)** | The question to ask (positional argument — no `--` needed) |
| `--top-k` | `3` | Number of chunks to retrieve as context |
| `--model` | `gpt-4o-mini` | OpenAI model to use for generating answers |
| `--collection` | `rag-basics` | ChromaDB collection name to search |
| `--persist-dir` | `./vectordb_data/chromadb` | Where ChromaDB data is stored |

---

## Step 3: Try Different Queries

Test with questions that span multiple documents:

```bash
# Single-topic question (pulls from one file)
python -m rag_beginner_basics query "What are the benefits of cloud computing?"

# Cross-topic question (pulls from multiple files)
python -m rag_beginner_basics query "What is the relationship between AI, machine learning, and Python?"

# Question not in your data (tests the honesty guardrail)
python -m rag_beginner_basics query "What is the capital of France?"
```

### What to look for:

| Query Type | Expected Behavior |
|------------|-------------------|
| **Single-topic** | Sources should all come from one file (e.g., `cloud_computing.txt`) |
| **Cross-topic** | Sources should come from multiple files (e.g., `artificial_intelligence.txt`, `machine_learning.txt`, `python_programming.txt`) |
| **Not in data** | The LLM should say "I don't have enough information to answer this" |

## Troubleshooting

| Error | Cause | Fix |
|-------|-------|-----|
| `No module named rag_beginner_basics.__main__` | Missing `__main__.py` | Create `rag_beginner_basics/__main__.py` (see Step 28b in TRAINING_GUIDE) |
| `ModuleNotFoundError: No module named 'pypdf'` | Wrong Python environment | Run `source .venv/bin/activate` or use `uv run python -m ...` |
| `openai.OpenAIError: The api_key client option must be set` | Missing API key | Create `.env` file with `OPENAI_API_KEY=sk-your-key` |
| `error: the following arguments are required: --data-dir` | Forgot `--data-dir` flag | Add `--data-dir ./data/sample_texts` to the ingest command |

---

## Alternative: Using `uv run`

If you don't want to activate the virtual environment every time, prefix commands with `uv run`:

```bash
uv run python -m rag_beginner_basics ingest --data-dir ./data/sample_texts
uv run python -m rag_beginner_basics query "What is machine learning?"
```

`uv run` automatically uses the correct `.venv` Python without needing `source .venv/bin/activate`.
